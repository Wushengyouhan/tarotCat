# 部署说明（阿里云 ACR + 服务器 Docker）

## 1. GitHub Secrets（你已在仓库里配好的几项）

构建工作流使用：

| Secret | 含义 |
|--------|------|
| `ACR_REGISTRY` | 仓库地址，**不要**带 `https://`，例：`crpi-xxxx.cn-shanghai.personal.cr.aliyuncs.com` |
| `ACR_USERNAME` | ACR 登录用户名 |
| `ACR_PASSWORD` | ACR 登录密码 |
| `ACR_NAMESPACE` | 命名空间，例：`wushengyouhan` |
| `ACR_REPOSITORY` | 镜像仓库名，例：`tarotcat` |

推送完整镜像名为：

`$ACR_REGISTRY/$ACR_NAMESPACE/$ACR_REPOSITORY:标签`

CI 会打两个标签：`latest` + 当前提交的 7 位短 SHA。

## 2. 推送代码自动构建

- 工作流：`.github/workflows/docker-acr.yml`
- 触发：`main` 分支有 **push** 时构建并推送镜像。

合并进 `main` 后，在 Actions 里看是否成功；成功后到 ACR 控制台确认镜像。

## 3. 服务器拉取并运行

在服务器上（已安装 Docker / Compose）：

```bash
# 登录 ACR（用户名、密码为阿里云控制台对应账号，勿写进仓库）
docker login crpi-wzkmn99umn8syi28.cn-shanghai.personal.cr.aliyuncs.com
```

编辑 **`docker-compose.yml`**：将 `SILICONFLOW_API_KEY` 的占位符 `REPLACE_WITH_YOUR_SILICONFLOW_API_KEY` 换成你的硅基流动 Key（其余镜像、端口已写死：**3001→3000**）。

镜像为 CI 推送的 **`latest`**：

`crpi-wzkmn99umn8syi28.cn-shanghai.personal.cr.aliyuncs.com/wushengyouhan/tarotcat:latest`

拉取并启动（先登录 ACR）：

```bash
docker compose pull
docker compose up -d
```

浏览器访问：`http://服务器IP:3001`。

**安全提醒**：密钥写在 yml 里后，**不要把含真 Key 的文件 `git push` 到公开仓库**；需要改镜像 tag 时，直接编辑 yml 里 `image:` 那一行即可。

## 4. 仓库里有哪些 Compose 文件

| 文件 | 用途 |
|------|------|
| `docker-compose.example.yml` | 与 `docker-compose.yml` 同结构，便于复制 |
| `docker-compose.yml` | 服务器模板：镜像、端口、**硅基变量占位符**（本地替换 Key） |
| `docker-compose_参考.yml` | **仅本地备忘**，且已列入 `.gitignore`，勿提交 |

可选：仍可用 **`.env.deploy.example`** 对照变量名；部署可不依赖 `.env` 文件。

## 5. 镜像构建

- `Dockerfile`：多阶段构建，Next.js `output: "standalone"`
- `.dockerignore`：减小构建上下文，排除 `node_modules`、测试与本地密钥文件

本地验证镜像（可选）：

```bash
docker build -t tarot-pwa:local .
docker run --rm -p 3001:3000 --env-file .env.deploy.example tarot-pwa:local
```

（`.env.deploy.example` 里是占位 key，AI 接口会失败，仅验证进程能起来。）
